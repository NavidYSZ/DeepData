# Universal Module-View-Pattern

Wird in Phase 2 erarbeitet. **Eine** Anatomie pro Modul-Page, dass alle Module folgen: PageHeader (Titel + Actions) → FilterBar (Date-Range, Source-Filter, etc.) → SectionCard(s) (KPIs, Charts, Tabellen) → StatsRow (Footer-Hinweise). Inspiration aus v1-[`page-shell.tsx`](../../components/dashboard/page-shell.tsx), aber konsequent durchgesetzt.

Ziel: User erkennt jedes Modul auf den ersten Blick wieder; Modul-Bauen wird zur Füll-Aufgabe statt Layout-Erfindung.

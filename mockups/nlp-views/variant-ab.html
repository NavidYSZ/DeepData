Was diese Variante kann (und was nicht)

✓ Echter dreistufiger Baum, der zeigt was alles unter welchem Subtopic abgedeckt ist.

✓ Subtopics ohne Entities ("nur erwähnt") fallen sofort auf — Indikator für oberflächliche Coverage.

✓ Aus den Entity-Listen pro Subtopic siehst du, was die Tiefe einer potenziellen Cluster-Page ausmachen würde.

✗ Sagt dir nicht: "und so sollte deine URL-Struktur aussehen" oder "diese Page solltest du neu anlegen". Das ergänzt Variante A+B+C.

! Prompt-Erweiterung nötig: Phase 3 bekommt belongs_to_subtopic: string | null. Resolver-Trick für Namens-Matching (analog canonical_name).

Topical-Map-Varianten

Vier Mockups, alle mit dem gleichen Beispiel (Sabah-Dentalmedizin Tornesch), damit du sie 1:1 vergleichen kannst. Sie sind keine echte Implementierung — nur visuelle Designs, um zu entscheiden welche Richtung wir umsetzen.

EINSTIEG no-risk · keine Prompt-Änderung

Variante A — Topical-Map-Tree

Baum aus den schon vorhandenen Phase-5-Daten (pillar_topic, subtopics, content_gaps, related_clusters). Pillar oben, Subtopics als Branches darunter, Gaps und Cluster als seitliche Listen.

Aufwand: 1 neue View-Komponente, keine Pipeline-Änderung · Risiko: minimal
→
MITTEL neues Prompt-Feld · 3-Ebenen-Baum

Variante A + B — Tree mit Entity-Zuordnung

Wie A, plus jede Entity bekommt belongs_to_subtopic. Drei Ebenen: Pillar → Subtopic → Entity. Jeder Subtopic-Knoten zeigt darunter die zugehörigen Entities.

Aufwand: Prompt-Erweiterung + Resolver für Subtopic-Strings · neue View · Risiko: LLM-Konsistenz beim Subtopic-Matching
→
VOLL neue Phase 6 · echte Sitemap-Empfehlung

Variante A + B + C — Empfohlene Sitemap

Macht aus dem Beschreibungstool ein Empfehlungstool: pro empfohlener Page bekommen wir H1, URL-Slug, status (covered_on_page / content_gap / likely_exists_elsewhere), und target-Queries. Genau die "wie sollte meine Sitemap aussehen"-Sicht.

Aufwand: neue Phase 6 im Prompt + neues Schema-Feld + neue View · Risiko: LLM hypothetisiert URL-Slugs — UI muss das transparent kommunizieren
→
GEZIELT kein neues View-Konzept · fixt nur das Hub/Child-Problem

Variante D — Parent-Pillar Hypothese

Statt einer neuen View: bei Child-Page-Erkennung wird im Page Profile zusätzlich der hypothetische Parent-Pillar und die Geschwister-Pages projiziert. Behebt das 3-Pillar-Dreieck-Problem direkt im bestehenden UI, ohne ein neues Map-Konzept.

Aufwand: Prompt-Erweiterung um parent_pillar_hypothesis · kleines Update im Page Profile · Risiko: Hypothesen werden als Fakten gelesen
→
Meine Empfehlung

Schau dir A zuerst an — wenn dich der reine Baum schon emotional packt, gehen wir A → A+B → A+B+C inkrementell. Falls dich A enttäuscht aber das 3-Pillar-Dreieck stört, ist D der gezielte Fix mit dem kleinsten Eingriff. Optionen sind nicht exklusiv — D kannst du auch zusätzlich zu A/A+B/A+B+C bauen.

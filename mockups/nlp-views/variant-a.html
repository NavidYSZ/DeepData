Was diese Variante kann (und was nicht)

✓ Zeigt auf einen Blick: ein Pillar-Topic, welche Subtopics auf der Seite behandelt werden, welche fehlen, welche Cluster nebenan liegen.

✓ Heute machbar, kein LLM-Eingriff, kein neues Schema-Feld nötig.

✗ Die Subtopics sind nackte Strings — du siehst nicht, welche Entities (z.B. "All-on-4", "Veneers") unter welchem Subtopic verortet sind. Das ergänzt Variante A+B.

✗ Keine Empfehlung für URL-Struktur / H1 / Page-Anzahl. Das ergänzt Variante A+B+C.

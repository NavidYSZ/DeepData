UI Redesign · DeepData GSC Dashboard

Drei Designrichtungen für ein moderneres DeepData.

Jedes Mockup adressiert die gleichen Kernprobleme — flache Hierarchie, uneinheitliche Filter, fehlende visuelle Tiefe — mit einer eigenen Designsprache. Wähle einen Stil, oder kombiniere Elemente.

Was im Audit aufgefallen ist

1. Filter sind nicht einheitlich. Kannibalisierung nutzt einen vollen Kalender-Picker, der Rest nutzt MonthPresetRangePicker. Manche Seiten zeigen Advanced-Filter inline, andere im Popover.
2. Visuell zu flach. Nur shadow-sm auf allen Cards, kaum Background-Differenzierung, alle Elemente auf der gleichen "Ebene".
3. Stats-Hierarchie fehlt. Die wichtigste Zahl sieht aus wie jede andere. KPIs verstecken sich im Fließtext.
4. Filter-Redundanz. "Min Impressions" gibt es in drei Varianten (Slider, Input, Tolerance). Drop-downs für Top N, Bucket, Mode überall verstreut.
01 Sanfte Evolution

Refined Depth

Bleibt nah am bestehenden Look, aber mit echter visueller Hierarchie: mehrstufige Schatten, abgesetzte Cards, abgerundete Ecken. Niedrigstes Risiko, größter Komfortgewinn.

02 Pro / Dense

Linear Pro

Dunkle Sidebar, helle Content-Area, hohe Daten-Dichte. Filter werden zu Chips mit "Add filter"-Pattern wie in Linear/Notion. Scharf, schnell, datenfokussiert.

03 Bold / Vibrant

Bento Studio

Farb-codierte KPI-Kacheln in einem Bento-Grid (Klicks blau, Impressionen lila, CTR grün — bereits im Code angelegt). Mehr Persönlichkeit, mehr Visualisierung, modernes Analytics-Tool-Feeling.

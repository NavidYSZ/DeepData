Was sich konkret ändert

Komponenten

  • → MonthPresetRangePicker wird zum Standard auf allen Seiten (Kannibalisierung migriert)
  • → DateRangePicker wird intern als Fallback hinter "Eigene…" gehalten
  • → Neue FilterChip Komponente (Label + Value + X)
  • → Neue SegmentedToggle für Mode-Picker (Query/Page, Alle/2/3-4/5+)
  • → Refactor FilterBar: Primary-Row + Active-Chips-Row + Advanced-Popover

Aufgeräumt / entfernt

  • − "Mehr" Dropdown beim Date Picker (jetzt Teil des Segmented Controls)
  • − "Top N" Dropdown auf Kannibalisierung & by Site (durch echte Pagination ersetzt)
  • − Axis-Fixed/Dynamic auf by Query (wandert zu Chart-Controls)
  • − Pos/CTR Window auf Position-vs-CTR (wandert zu Chart-Zoom)
  • − Min-Impressions-Toggle (Auto/Custom) — wird zum Slider im Erweitert-Popover
Implementierungs-Reihenfolge (wenn du grünes Licht gibst):
  1. 1. Design-Tokens (Farben, Schatten, Radius) in globals.css migrieren
  2. 2. Neue FilterChip, SegmentedToggle, FilterBarV2 in components/ui/
  3. 3. MonthPresetRangePicker als Segmented Control neu bauen
  4. 4. Pro Seite migrieren: Kannibalisierung → Data Explorer → Top Mover → Rank Tracker → URL Tracker → SEO Bubble
  5. 5. Eine Seite (z. B. Kannibalisierung) zuerst als Pilot, dann iterieren bevor der Rest folgt
